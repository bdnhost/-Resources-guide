📊 Power BI Masterclass

הפכו נתונים יבשים לתובנות עסקיות ויזואליות

למה Power BI?

Power BI של מיקרוסופט הוא הכלי המוביל בעולם לבינה עסקית (Business Intelligence). הוא מאפשר לכם לקחת נתונים מאקסל, SQL, והאינטרנט - ולאחד אותם לדשבורד אינטראקטיבי אחד שמתעדכן בזמן אמת.

חיבור לכל מקור מידע

קחו נתונים מ-Excel, SQL Server, Google Analytics ועוד מאות מקורות בלחיצת כפתור.

אינטראקטיביות

לחיצה על גרף אחד מסננת אוטומטית את כל שאר הדשבורד. חקרו את הנתונים לעומק.

DAX

שפת נוסחאות חזקה (דומה לאקסל אך חזקה פי כמה) לחישוב מדדים עסקיים מורכבים.

תהליך העבודה: מ-Data ל-Dashboard

  • Get Data (ייבוא נתונים): חיבור למקורות המידע וטעינת הטבלאות למודל.
  • Transform (עיבוד ב-Power Query): ניקוי הנתונים. מחיקת שורות ריקות, שינוי סוגי עמודות, ואיחוד טבלאות. זה השלב הקריטי ביותר.
  • Model (בניית מודל): יצירת קשרים (Relationships) בין הטבלאות. בדרך כלל במבנה "כוכב" (טבלת עובדות במרכז, טבלאות מימדים מסביב).
  • Visualize (ויזואליזציה): גרירת גרפים, מפות וטבלאות לקנבס. עיצוב הצבעים והפונטים.
  • Publish (פרסום): העלאת הדוח לענן (Power BI Service) ושיתוף עם מנהלים או לקוחות.

🎨 איזה גרף לבחור? (Visualizations Guide)

בחירה לא נכונה של גרף יכולה להרוס תובנה טובה. הנה המדריך המהיר:

📊 השוואה (Comparison)

מתי להשתמש: כשרוצים להשוות ערכים בין קטגוריות.

  • Bar Chart: כשיש שמות קטגוריות ארוכים.
  • Column Chart: להשוואת מעט קטגוריות (עד 8).

📈 מגמה (Trend)

מתי להשתמש: כשרוצים לראות שינוי לאורך זמן (Time Series).

  • Line Chart: הבחירה הקלאסית והברורה ביותר.
  • Area Chart: להדגשת הנפח מתחת לקו.

🍰 חלק מהשלם (Part-to-Whole)

מתי להשתמש: כשרוצים להראות התפלגות באחוזים.

  • Donut / Pie Chart: רק כשיש פחות מ-4 קטגוריות!
  • Treemap: כשיש הרבה קטגוריות והיררכיה.

🗺️ מיקום (Spatial)

מתי להשתמש: לנתונים גיאוגרפיים.

  • Map (Bubbles): גודל העיגול מייצג את הערך.
  • Filled Map: צביעת מדינות/איזורים לפי חומרת המדד.

נוסחאות DAX מתקדמות

Data Analysis Expressions - השפה שמאחורי המנוע של Power BI.

Time Intelligence (בינה של זמן)

חישוב השוואות בין תקופות באופן אוטומטי.

Sales YTD (מתחילת שנה עד היום):
Sales YTD = TOTALYTD(SUM(Sales[Amount]), 'Date'[Date])
Sales Growth % (צמיחה חודשית):
Sales MoM% =
VAR CurrentSales = [Total Sales]
VAR LastMonthSales = CALCULATE([Total Sales], PREVIOUSMONTH('Date'[Date]))
RETURN
DIVIDE(CurrentSales - LastMonthSales, LastMonthSales, 0)

Year Over Year (שינוי שנתי)

Sales YoY% =
VAR CurrentSales = [Total Sales]
VAR LastYearSales = CALCULATE([Total Sales], SAMEPERIODLASTYEAR('Date'[Date]))
RETURN
DIVIDE(CurrentSales - LastYearSales, LastYearSales)

💡 טיפ זהב

כדאי לדעת: DAX זה לא Excel! הוא עובד על עמודות שלמות, לא על תאים בודדים. חשבו "טבלאי" ולא "תאי"!

🎯 מסקנות ויישום מעשי

✅ מה למדנו היום?

  • Power BI מאחד הכל - מקורות נתונים שונים במקום אחד
  • תהליך מובנה - Get Data → Transform → Model → Visualize
  • בחירת גרף נכונה - חשובה לתקשורת יעילה
  • DAX לחישובים - מדדים עסקיים מתקדמים
  • אינטראקטיביות - דשבורדים חיים ומעדכנים

📊 איך להמשיך להתפתח?

  1. התחל עם Power BI Desktop - חינם מהאתר של מיקרוסופט
  2. השתמש בנתונים שלך - Excel או CSV קיימים
  3. למד Power Query - עיבוד נתונים מתקדם
  4. תרגל DAX - התחל עם SUM, COUNT, AVERAGE
  5. בנה דשבורד אמיתי - לעבודה או לתחביב

⚠️ טעויות נפוצות

  • יותר מדי גרפים - פחות זה יותר
  • צבעים לא עקביים - בלבל את הצופים
  • לא לנקות נתונים - Garbage In, Garbage Out
  • דשבורד איטי - יותר מדי נתונים בו-זמנית
  • לא לחשוב על המשתמש - מי יראה את זה?

❓ שאלות נפוצות

איך מתחברים למקור נתונים חיצוני?

לחצו על "Get Data" ובחרו את סוג המקור (SQL Server, Excel, Web וכו'). תזדקקו לפרטי חיבור כמו שם שרת וסיסמה.

מה עושים כשהדשבורד איטי?

בדקו את כמות הנתונים, השתמשו בסינונים, והימנעו מחישובים מורכבים מדי. לפעמים כדאי לסכם נתונים מראש.

איך משתפים דשבורד עם אחרים?

פרסמו ל-Power BI Service (הענן) ושתפו קישור או הזמינו משתמשים. זקוקים ללרישיון Power BI Pro או Premium.
