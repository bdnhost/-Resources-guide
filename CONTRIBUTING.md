## 🤝 קוד התנהגות

### העקרונות שלנו

- **כבוד הדדי** - נתייחס לכולם בכבוד ובהערכה
- **פתיחות** - נקבל בברכה נקודות מבט שונות

✅ שימוש בשפה מכבדת וכוללת

❌ שפה או דימויים מיניים

## ✍️ הנחיות כתיבה

### עקרונות כתיבה

1. **בהירות** - כתוב באופן פשוט ומובן
2. **תמציתיות** - הגע לעניין במהירות
3. **מבנה** - ארגן את התוכן בצורה הגיונית
4. **דוגמאות** - הוסף דוגמאות מעשיות
5. **ויזואליזציה** - השתמש בתמונות ותרשימים

### סגנון כתיבה

- **גוף שני יחיד** - "אתה תלמד..." (לא "תלמדו" או "נלמד")
- **פעיל על פני סביל** - "צור קובץ" (לא "קובץ נוצר")
- **משפטים קצרים** - 15-20 מילים למשפט
- **כותרות תיאוריות** - "התקנת Python" (לא "התקנה")

### טון

- ידידותי אך מקצועי
- מעודד ומעצים
- ללא ז'רגון מיותר
- נגיש למתחילים

## 📐 מבנה מדריך

### תבנית מדריך בסיסי

```html
<!DOCTYPE html>
<html lang="he" dir="rtl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>שם המדריך - LearningHub</title>
    <meta name="description" content="תיאור קצר של המדריך">

    <!-- SEO Meta Tags -->
    <meta property="og:title" content="שם המדריך">
    <meta property="og:description" content="תיאור המדריך">
    <meta property="og:type" content="article">

    <link rel="stylesheet" href="../public/global.css">
</head>
<body>
    <!-- Header -->
    <header>
        <h1>שם המדריך</h1>
    </header>

    <!-- Main Content -->
    <main>
        <!-- מבוא -->
        <section id="intro">
            <h2>מבוא</h2>
            <p>תיאור קצר של מה המדריך מכסה ולמי הוא מיועד</p>
        </section>

        <!-- תוכן עיקרי -->
        <section id="content">
            <h2>כותרת ראשית</h2>
            <p>תוכן...</p>

            <h3>כותרת משנה</h3>
            <p>תוכן...</p>
        </section>

        <!-- סיכום -->
        <section id="summary">
            <h2>סיכום</h2>
            <p>נקודות מרכזיות שנלמדו</p>
        </section>
    </main>

    <script src="../public/global.js"></script>
</body>
</html>
```

### מבנה תוכן מומלץ

1. **מבוא** (10%)
   - מהו הנושא
   - למי המדריך מיועד
   - מה תלמד

2. **תוכן עיקרי** (80%)
   - חלוקה לסקציות ברורות
   - התקדמות מפשוט למורכב
   - דוגמאות קוד/תמונות

3. **סיכום** (10%)
   - נקודות מפתח
   - משאבים נוספים
   - שלבים הבאים

## 🔄 תהליך הגשה

### שלב 1: הכנה

```bash
# Fork את המאגר דרך GitHub

# Clone את ה-fork שלך
git clone https://github.com/YOUR-USERNAME/-Resources-guide.git

# הוסף remote למאגר המקורי
git remote add upstream https://github.com/bdnhost/-Resources-guide.git

# צור branch חדש
git checkout -b feature/your-guide-name
```

### שלב 2: כתיבה

1. צור קובץ HTML חדש בתיקייה המתאימה:
   - `guides/ai-automation/` - מדריכי AI
   - `guides/career/` - קריירה
   - `guides/creative-studio/` - עיצוב
   - `guides/data-business/` - נתונים
   - `guides/digital-basics/` - בסיס
   - `guides/technology/` - טכנולוגיה

2. שם הקובץ:
   - השתמש באותיות קטנות
   - מקפים במקום רווחים
   - תיאורי: `chatgpt-advanced-prompts.html`

3. כתוב את התוכן לפי התבנית

### שלב 3: בדיקה

```bash
# פתח את הקובץ בדפדפן ובדוק:
# ✓ התוכן מוצג נכון
# ✓ קישורים עובדים
# ✓ תמונות נטענות
# ✓ Responsive על מובייל
```

### שלב 4: Commit

```bash
# הוסף את הקבצים
git add .

# Commit עם הודעה תיאורית
git commit -m "Add: מדריך מקיף לשימוש ב-ChatGPT"

# Push ל-fork שלך
git push origin feature/your-guide-name
```

### שלב 5: Pull Request

1. עבור ל-GitHub
2. לחץ על "New Pull Request"
3. מלא את הטופס:
   - **כותרת**: תיאור קצר וברור
   - **תיאור**: מה הוספת/שינית ולמה

#### תבנית Pull Request

```markdown
## סוג השינוי
- [ ] מדריך חדש
- [ ] שיפור מדריך קיים
- [ ] תיקון באג
- [ ] עדכון תיעוד

## תיאור
תיאור קצר של השינוי שלך

## קטגוריה

- [ ] Career
- [ ] Creative Studio
- [ ] Data & Business

## Checklist
- [ ] בדקתי שהקוד עובד
- [ ] בדקתי responsive

- [ ] הוספתי meta tags
```

## 🎨 סטנדרטים טכניים

### HTML

```html
<!-- טוב ✓ -->
<section id="intro">
    <h2>מבוא</h2>
    <p>תוכן...</p>
</section>

<!-- לא טוב ✗ -->
<div class="intro">
    <h2>מבוא</h2>
    <p>תוכן...</p>
</div>
```

- השתמש בתגיות סמנטיות (semantic HTML)
- `id` לסקציות ראשיות
- `class` לעיצוב
- ARIA labels לנגישות

### CSS

```css
/* טוב ✓ */
.card {
    padding: 1.5rem;
    border-radius: 8px;
}

/* לא טוב ✗ */
.card {
    padding: 24px;
    border-radius: 8px;
}
```

- השתמש במשתני CSS מ-`global.css`
- יחידות: `rem` (לא `px`)
- שמות מתארים: `.btn-primary` (לא `.blue-button`)

### JavaScript

```javascript
// טוב ✓
const handleClick = () => {
    console.log('Clicked');
};

// לא טוב ✗
function handleClick() {
    console.log('Clicked');
}
```

- ES6+ syntax
- Arrow functions
- `const`/`let` (לא `var`)

### קבצים

```
guides/
└── ai-automation/
    ├── chatgpt-basics.html       ✓
    ├── prompt-engineering.html   ✓
    └── ChatGPT_Guide.html        ✗ (אותיות גדולות)
```

- שמות קבצים: `lowercase-with-dashes.html`
- תמונות: `assets/images/guide-name/image.png`
- אודיו: `assets/audio/guide-name/audio.mp3`
